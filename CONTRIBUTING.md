# Как внести вклад в TechRole Index

TechRole Index принимает исправления кода, методологии, документации и проверяемых данных. Приоритет — воспроизводимость и честное происхождение показателей, а не увеличение числа строк любой ценой.

## До начала работы

1. Для ошибки данных укажите профессию, регион/уровень, период, публичный URL источника и ожидаемое поведение. Не публикуйте контакты соискателей, работодателей или другие персональные данные.
2. Для нового источника приложите официальную документацию API, условия использования и описание rate limits. HTML scraping, CAPTCHA/proxy обходы и скрытая ротация аккаунтов не принимаются.
3. HH нельзя включать без всех guard-переменных и отдельно подтверждённого права коммерческого использования. Наличие provider-кода не является таким разрешением.
4. Не смешивайте gross, net и неизвестный налоговый статус. Недостаточная выборка должна оставаться `null`/«Недостаточно данных», а не превращаться в ноль или синтетическую оценку.

## Локальная проверка

```powershell
docker compose up --build -d
docker compose exec -T backend sh -lc "ruff check . && mypy app && pytest"
docker compose exec -T frontend sh -lc "npm run lint && npm run typecheck && npm test"
docker compose run --rm --no-deps -e NEXT_DIST_DIR=.next-contribution-check frontend npm run build
docker compose config --quiet
```

Если меняются публичные страницы или machine endpoints, дополнительно запустите Chromium E2E и sitemap audit:

```powershell
docker compose --profile e2e run --rm e2e
docker compose exec -T frontend npm run audit:public
```

## Требования к pull request

- изменение решает одну понятную задачу и не переформатирует посторонние файлы;
- новая логика покрыта тестом, а важное предположение записано в `DECISIONS.md`;
- публичные profession pages остаются SSR и индексируемыми;
- Premium-поля не попадают в free JSON, HTML, JSON-LD или экспорты;
- schema/provenance/citation links обновлены вместе с изменением контракта данных;
- в commit нет `.env`, dump, токенов, паролей, приватных URL и персональных данных;
- обязательные команды из `AGENTS.md` прошли.

## Безопасность

Не создавайте публичный issue для найденного секрета или уязвимости, которую можно эксплуатировать. Используйте приватный контакт из `SECURITY.md`. Не прикладывайте production logs целиком: сначала удалите cookies, email, IP, токены и содержимое заявок.

Отправка изменений не меняет права на сторонние данные и не означает автоматического принятия. Лицензия проекта будет явно зафиксирована владельцем отдельно; до этого публичная доступность репозитория сама по себе не даёт дополнительных прав на переиспользование.
