# Методология TechRole Index v1.1.0

## Salary

Срез: профессия × seniority × region × период × gross/net basis.

- salary coverage = вакансии с хотя бы одной границей / все вакансии;
- midpoint = `(from + to) / 2` только при двух границах;
- медиана, среднее, P25/P75 считаются по midpoint;
- медианы `from` и `to` считаются независимо по доступным границам;
- одна граница не используется для восстановления другой;
- gross/net/unknown не смешиваются;
- валюта нормализуется отдельным датированным `CurrencyRateProvider`, исходные значения сохраняются;
- при `n < MIN_SALARY_SAMPLE` (3 по умолчанию) midpoint-показатели не публикуются.

Confidence: `insufficient` при n ниже порога; `low` при n < 2×порог или coverage <35%; `medium` при n <5×порог или coverage <60%; иначе `high`.

### Вилки из официальных публикаций

Публикации «Работы России» не смешиваются с основной gross-витриной, потому что источник не задаёт совместимый gross/net-признак. Для каждого уровня Junior/Middle/Senior рассчитываются отдельные RUB-срезы за 180 дней. Median, average и P25/P75 публикуются только по midpoint полных вилок и только при `n ≥ 3`. Одиночные границы учитываются в salary coverage, но не превращаются в выдуманную полную вилку. Отдельный недельный график полноты показывает число новых публикаций, число полных RUB-вилок и их долю; он объясняет, почему большой поток публикаций не означает большую зарплатную выборку. При менее чем 20 точных публикациях график полноты явно переключается на направление. Публикации на основном графике складываются по неделям; точная профессия и всё направление остаются двумя разными рядами.

История зарплаты раз в неделю показывает накопительную медиану от начала периода и обязательно включает последний день. Для каждого уровня сохраняется наблюдаемый ряд точной профессии, а если он не образует хотя бы трёх видимых точек — более наполненный ряд направления с явной подписью. Недостающий уровень может быть показан пунктирной горизонтальной линией из одного непротиворечивого открытого исследования; это статичный ориентир, а не историческое наблюдение. Наблюдаемые линии не сортируются и не «исправляются»: пересечение Middle/Senior возможно из-за разного состава малых выборок и сопровождается объяснением. Карточки уровней используют либо полный неубывающий официальный комплект, либо один полный неубывающий комплект одного исследования. Переключатель 30/90/180 дней меняет только видимое окно уже опубликованного 180-дневного ряда и не пересчитывает значения.

График потока публикаций выбирает один охват, а не накладывает несопоставимые ряды: точную профессию при 20 и более публикациях, иначе явно подписанное направление. Показываются недельные суммы и четырёхнедельная скользящая средняя. График полноты использует тот же порог выбора охвата. Это обеспечивает заполненный контекст для редких ролей, не выдавая направление за точную профессию.

### Публичные ориентиры фактических доходов

Это третий независимый слой, а не способ заполнить NULL в vacancy metrics. Базовый публичный срез — исследование Хабр Карьеры за I полугодие 2026 (`n=45 226`). Оно основано на анонимно указанных текущих доходах специалистов; диаграммы калькулятора показывают общий доход как сумму медианного оклада и медианной премии. Серия отчётов использует суммы на руки.

Дополнительно для каждой профессии показывается официальный широкий фон Росстата по форме № 57-Т за октябрь 2025 года: средняя начисленная зарплата до вычета налогов по составной группе ОКЗ. Используются три опубликованных значения таблицы 9: разработчики и аналитики ПО и приложений — 216 348 ₽ (оценка 158,4 тыс. работников), специалисты по базам данных и сетям — 165 966 ₽ (62,9 тыс.), специалисты высшего уровня в ИКТ — 202 029 ₽ (221,3 тыс.). Число работников — оценка охваченной группы, не размер выборки. Этот `occupation_group` всегда подписан как широкая группа, не является медианой и не переименовывается в зарплату конкретной роли.

Для восьми профессий без прямого значения в общем отчёте добавлен снимок на 22 июля 2026 года из публичного SEO-описания отфильтрованной страницы калькулятора Хабр Карьеры: ручной тестировщик, Data Scientist, MLOps, Computer Vision, специалист и инженер по информационной безопасности, SOC-аналитик и пентестер. Сохраняется только открыто показанная медиана. Расширенные распределения и грейды, скрытые входом в аккаунт, не извлекаются; размер выборки и gross/net у этих точек остаются `unknown`. Живая страница может измениться, поэтому число хранится как датированный снимок, а не автоматически обновляемый факт. Опциональная ночная проверка повторно читает только публичные метаданные: совпадение подтверждает снимок, HTTP-ошибка означает временную недоступность, а новое число требует ручного решения и новой даты версии.

Для каждой профессии сохраняются `source_id`, период, дата публикации, валюта, tax status, география, тип метрики, scope и пояснение сопоставления. Приоритеты:

1. точная профессия;
2. данные по технологии либо явно смежная профессия;
3. широкая группа занятий как независимый официальный фон;
4. категорийный fallback.

P10 и P90 описывают распределение доходов и не интерпретируются как Junior/Senior. Грейдовые значения добавляются только из источников, где грейд указан явно. Малая выборка не скрывается: например, значения Grades/GetGrade сохраняют собственный `n`, tax status `unknown` и предупреждение об отмеченном авторами выбросе .NET. Для каждого разработческого SKU профессии хранится полный резервный комплект грейдов GetGrade, для остальных ролей — полный комплект общерыночных медиан опроса Профсоюза работников ИТ (`n=1 539`, Junior 114 500 ₽, Middle 200 000 ₽, Senior 310 000 ₽). Резерв подписывается как общий рынок и выбирается целиком только тогда, когда более точный комплект неполон или пересекается. Так заполнены 150 из 150 грейдовых карточек; покрытие основного ролевого ориентира остаётся 37 прямых и 13 смежных срезов.

## Trends

Для N ∈ {7, 30, 90}: `Δ% = (avg(current N) - avg(previous N)) / avg(previous N) × 100`.

- up: > +3%;
- neutral: −3%…+3% включительно;
- down: < −3%;
- unknown: нет обоих окон или предыдущий average равен нулю.

## Score 0–100

Версия `v1.1.0`. Веса: demand 28%, salary 24%, demand growth 16%, junior access 12%, remote share 10%, data quality 10%.

Demand проходит `log1p`, salary/growth ограничиваются по 5/95 перцентилю. Эти три компонента переводятся в percentile rank среди активных профессий. Junior share насыщается при 35%, remote ограничивается 0…1. Data quality = `0.7 × coverage + 0.3 × min(sample/100, 1)`.

Компонент salary больше не берётся из подготовленной demo-витрины. Для каждой профессии выбирается одно публичное воспроизводимое значение: национальный точный срез, затем данные по технологии, смежная роль, медиана доступных ролевых регионов, широкая группа занятий Росстата и только затем категорийный ориентир. Грейдовые точки не смешиваются в этот компонент. Demand, growth, junior access, remote share и data quality остаются показателями подготовленной сопоставимой витрины, что прямо отмечено в интерфейсе.

`Score = 100 × Σ(weight × component)`, результат ограничивается 0…100. На странице профессии показывается не только оценка каждого фактора, но и его вклад: например, `70/100 × 30% = +21 балл`. Сумма вкладов равна итоговому индексу с учётом округления. Сохраняются version, breakdown и data confidence. Новые веса создают новую `scoring_versions` запись.

## Навыки и регионы

Число у навыка — количество записей подготовленной витрины этой профессии, где он указан. Число у региона — расчётный объём последнего подготовленного среза. Это два разных счётчика: первый не является рейтингом навыков, второй не является текущим остатком вакансий региона.

## Classification

Unicode/whitespace normalization → alias dictionary → regex → exclusion rules → experience → RU/EN seniority markers → confidence. В `rules-v2` признаки технологий применяются только к обобщённым software-заголовкам: результат принимается, если после подавления более специфичных ролей остался ровно один однозначный сигнал. Явный алиас заголовка всегда приоритетнее skills, а неоднозначность приводит к abstention. Lead/principal/architect исключаются из автоматического Senior. Optional AI применяется только после низкой уверенности rule-based результата и получает верхний confidence 0.79.

Повторная классификация меняет только строки без решения или с `rules-*`; AI- и ручные решения не перезаписываются. Команда сначала поддерживает dry-run, затем использует тот же детерминированный классификатор при применении.

## Ограничения

Вакансии не равны фактическим наймам; опубликованная вилка не равна офферу; фактический доход участника исследования не равен будущему офферу; coverage может быть смещён по профессии/региону/источнику. Основная воспроизводимая витрина пригодна для проверки продукта и не должна выдаваться за измерение рынка. Официальные публикации и публичные зарплатные исследования показываются отдельными provenance-слоями с собственными ограничениями.
