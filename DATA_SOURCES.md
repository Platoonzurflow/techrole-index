# Источники данных

## DemoVacancyProvider

Включён по умолчанию. Генерирует детерминированные вакансии, валюты, пропуски и seniority без реальных людей/работодателей. Применим только для разработки, демонстрации и тестов.

## HhApiProvider

Заготовка использует только официальный API endpoint, максимум одну страницу/100 записей за вызов и обязательный `HH-User-Agent: App/Version (contact)`. Provider не делает HTML scraping, не обходит CAPTCHA, rate limit или глубину выдачи, не использует прокси-ротацию и не содержит логотипов/товарных знаков.

Он выключен до одновременного выполнения guards: `HH_ENABLED=true`, `HH_COMMERCIAL_USE_CONFIRMED=true`, app name, contact email. Технический флаг не заменяет письменное юридическое разрешение. Коммерческое использование данных API может требовать отдельного разрешения правообладателя.

## TrudvsemOpenDataProvider

Реализован provider официального открытого JSON API портала «Работа России»: `https://opendata.trudvsem.ru/api/v1/vacancies`. Описание параметров и условий находится на официальной странице `https://trudvsem.ru/opendata/api`. HTML-страницы не извлекаются.

Страница наборов `https://trudvsem.ru/opendata/datasets` прямо относит публикацию к открытым данным Роструда, ссылается на федеральные типовые условия и требует ссылку на `trudvsem.ru` при копировании. Поскольку прежний `data.gov.ru/information-usage` сейчас отвечает 404, машинные metadata ссылаются на действующее официальное зеркало тех же типовых условий: `https://rospatent.gov.ru/ru/opendata/uslovia-od`. Это фиксирует применимые условия и attribution, но не выдаёт производную работу TechRole Index за CC BY и не заменяет production-проверку юристом.

Это самостоятельная общероссийская база портала Роструда, а не API или зеркало hh.ru. API сообщает об общем числе хранимых вакансий и отдаёт собственные source-поля; отдельные записи могут поступать от работодателей или иных интернет-ресурсов, поэтому происхождение конкретной записи сохраняется в `raw_payload`, но без доказательств она не приписывается hh.ru.

Проверка 2026-07-19 показала 538 722 доступные записи во всей базе на момент запроса. Полный разрешённый проход TechRole Index по 183 поисковым запросам и 258 страницам дал 9 293 ответа и 5 246 уникальных записей внутри 180-дневного окна. В актуальной БД с ранее полученными неповторяющимися записями находятся 5 253 публикации, из них 1 023 уверенно классифицированы в 33 из 50 профессий и 1 015 имеют хотя бы одну границу зарплаты. Для 17 узких профессий уверенных публикаций не нашлось: отсутствие оформляется как «Недостаточно данных», а не заполняется синтетически.

Безопасные ограничения MVP:

- только `GET` к разрешённому API endpoint;
- ограниченные `limit`, число профессий и задержка между запросами задаются окружением;
- уникальность сохраняется как `(trudvsem_open, external_id)`;
- raw payload строится по allowlist и исключает `contact_person`, `contact_list`, адреса и другие контактные поля;
- официальный ответ хранится с provenance source/run/query;
- поиск проходит по русскому/английскому названию и алиасам, результат дедуплицируется по source/external ID;
- API-параметр даты не используется как единственная гарантия: окно 180 дней повторно проверяется по `creation-date` на стороне ingestion;
- история показывает число публикаций по дате создания записи, а не восстановленное число одновременно активных вакансий;
- при неопределённой профессии может сработать локальная Ollama, но только после rule-based классификатора, с лимитом вызовов и confidence cap;
- поскольку API не даёт гарантированно совместимый gross/net-признак, `gross = unknown`: наблюдения не смешиваются с публичной gross-витриной и не запускают её перерасчёт.

Публичная зарплатная статистика официального слоя считается отдельно для Junior/Middle/Senior в RUB. Полные midpoint-показатели используют только записи с двумя границами, требуют минимум 20 наблюдений и не приписывают источнику неизвестный gross/net. Исторические точки используют скользящее 30-дневное окно с шагом семь дней. На момент проверки достаточная выборка есть только у части сочетаний профессия × уровень; остальные сочетания возвращают явный статус недостаточности.

Публичный `/data-status` и `/data-status.json` описывают подготовленную и официальную части как разные слои. Поле `current_market_claim` для обоих равно `false`: подготовленная дата не выдаётся за подтверждённую свежесть рынка, а исторические публикации не выдаются за одновременно активные вакансии. `/open-data.csv` разворачивает 50 профессий × 3 seniority в стабильные строки с периодом, размером выборки, confidence, налоговым статусом, canonical URL, источником и методологией.

Официальная документация HH, проверенная 2026-07-19, поддерживает `date_from`/`date_to`, зарплатные фильтры и страницы до 100 записей, но ограничивает глубину выдачи 2 000 результатами. Без авторизации после первого запроса может быть предложена CAPTCHA; отдельный salary-bank endpoint требует авторизации и оплаченного доступа к отчётам. Поэтому анонимный массовый обход не используется: для HH нужны официальный access token, согласованные права и все guard-переменные.

Provider включается `TRUDVSEM_ENABLED=true`. Условия и формат API нужно повторно проверить перед production, а retention и коммерческий сценарий - согласовать с владельцем данных/юристом.

## Добавление источника

Реализуйте `VacancyDataProvider.fetch`, сохраните provider terms URL, source/external ID, rate-limit policy и provenance. До production пройдите юридическую проверку прав, retention и требований attribution. Расчётные модули не должны зависеть от бренда источника.

## Currency

`DemoCurrencyRateProvider` намеренно фиксирован и продолжает обслуживать только воспроизводимый seed. `CbrCurrencyRateProvider` реализует отдельный реальный контур официального HTTPS XML Банка России `https://www.cbr.ru/scripts/XML_daily.asp`: передаёт `date_req`, учитывает nominal, сохраняет requested/effective date, источник и rate-to-RUB в `currency_rate_snapshots`. Выходной или праздник может вернуть последнюю установленную effective date, поэтому она не подменяется requested date.

Provider выключен по умолчанию через `CBR_CURRENCY_ENABLED=false`; на локальной машине владельца он включён и snapshot USD/EUR/KZT за requested date 2026-07-20 сохранён с effective date 2026-07-18. Dagster запускает currency op отдельно от vacancy ingestion. Наличие курса не разрешает автоматически преобразовывать или публиковать зарплату с неизвестным gross/net basis.
